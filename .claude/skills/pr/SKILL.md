# PR Skill
Создай pull request на GitHub, соблюдая соглашение проекта.

## Аргументы
- $0 - Название pull request по Conventional Commit (например `feat(web): добавить авторизацию`)
- $1 - Целевая ветка (по умолчанию `main`)

## Алгоритм

1. Определить целевую ветку: если $1 не передан — использовать `main`
2. Проверить, что текущая ветка не совпадает с целевой (`main` / `develop`)
3. Проверить отсутствие незакоммиченных изменений: `git status --porcelain`
   - Если есть незакоммиченные файлы — предупредить и прервать
4. Убедиться, что ветка запушена в remote: `git status -sb`
   - Если нет upstream — выполнить `git push -u origin HEAD`
   - Если есть непушеные коммиты — выполнить `git push`
5. Проверить, не существует ли уже открытый PR для этой ветки: `gh pr view --json url`
   - Если существует — вывести ссылку и завершить
6. Получить список коммитов: `git log <base>..HEAD --oneline`
7. Получить полный diff: `git diff <base>...HEAD`
8. На основе коммитов и diff составить тело PR по шаблону ниже
9. Создать PR: `gh pr create --title "$0" --base "<base>" --body "..."`
10. Вывести URL созданного PR

## Шаблон тела PR

```
## Summary
<Краткое описание изменений — 2-3 предложения>

### Реализованный функционал
- <Пункт 1>
- <Пункт 2>

### Технические детали
- <Архитектурные решения, важные зависимости>

## Test plan
- [ ] <Шаг проверки 1>
- [ ] <Шаг проверки 2>
```

## Правила составления описания

- Summary — что и зачем, не как
- Реализованный функционал — только видимые изменения (UI, endpoints, поведение)
- Технические детали — паттерны, архитектурные решения, нетривиальные зависимости
- Test plan — конкретные шаги для ручной проверки; не дублировать очевидное

## Обработка ошибок

| Ситуация | Действие |
|---|---|
| Текущая ветка = целевая | Прервать, сообщить |
| Есть незакоммиченные изменения | Прервать, вывести список файлов |
| PR уже существует | Вывести ссылку, не создавать новый |
| `gh` не авторизован | Сообщить `gh auth login` |
